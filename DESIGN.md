# 最后一次讲演 · Frontend Design Direction

## Intent

把当前页面从“教学后台面板”改成“中文社论式专题页面”。用户第一眼应该感到它像一张有纸感、有节奏、有文学气质的专题版面，而不是平铺卡片的工具页。

## Visual Language

- 整体气质：民国演讲现场 + 纸页 editorial layout
- 主字体：`Noto Serif SC` 用于标题、正文高光、分数等关键内容
- 辅助字体：`Manrope` 与 `Noto Sans SC` 用于标签、状态、按钮和信息层
- 主色：纸页米白、墨黑、赭红
- 辅色：暗金、墨绿，仅作状态与点缀，不抢主叙事
- 纹理：允许极轻的网格纸/印刷纹理，但不能像背景图堆料

## Layout Rules

- 顶部必须像专题头版，标题巨大，副标题克制，右侧是两到三张信息卡
- 左侧保留关卡导航，但做成“章节目录”而不是后台列表
- 中央内容区突出“当前段落”，段落文本必须像纸页正文，留白充足
- 录音与实时转写区域要像“直播评审台”，不能和正文抢戏
- 点评结果区域必须有一个极强的总分核心视觉，再往下展开维度和建议
- 历史记录使用时间轴或档案条目风格，而不是普通表格式列表

## Interaction Rules

- 选中的关卡必须明显，但不依赖高饱和色块
- 主按钮只保留一个真正的视觉焦点
- 空状态和错误状态都要保持“专题页面”的调性，不要突然变成管理系统提示框
- 移动端优先保留阅读顺序：标题 -> 当前关卡 -> 录音控制 -> 点评 -> 历史

## Avoid

- 不要大面积使用默认圆角毛玻璃卡片
- 不要紫色系、科技蓝系和通用仪表盘视觉
- 不要用过多渐变把页面刷得发亮
- 不要让所有模块看起来同一个权重
